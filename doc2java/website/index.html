doc2java

Use doc2java to create Java-source files from JavaDoc.

Setter- and getter-methods as well as variables will be created automatically. The documentation itself will also be copied into the source-files.

How to use

Just follow these simple steps:

  1. Download the doc2java.jar file.
  2. Call java -jar doc2java.jar -R mydocdir from the command line, where mydocdir is the path to the directory containing the JavaDoc-files.
  3. The code will be generated in the current directory.

Obviously you need to have a Java Runtime Environment (1.2 or higher) installed.

Options and parameters

You can give several command-line-paramters.

Usage is java -jar doc2java.jar [options] file

Options:

  • -R (or --recursive): scan all subdirectories as well.
  • -o (or --output): the directory where to write the java files, defaults to the current directory.
  • --notodo: write no todo tags into the files.
  • --todotag: use the given to-do-tag, defaults to "//TODO".

Parameters:

  • file: either the JavaDoc file or the directory containing the JavaDoc files, which should be processed.

Examples:

  • java -jar doc2java.jar MyClass.html
    creates the source code for the class decribed in MyClass.html.
  • java -jar doc2java.jar mydoc
    creates the source code for all classes described in the directory mydoc.
  • java -jar doc2java.jar -R mydoc
    creates the source code for all classes described in the directory mydoc and its subdirectories.
  • java -jar doc2java.jar -R --todotag "//TODO rob" mydoc
    same as above, but the generated todo-tags will now start with "//TODO rob".
  • java -jar doc2java.jar -R --notodotag mydoc
    same as above, but the no todo-tags will be generated.
  • java -jar doc2java.jar -R --output /home/myname/workspace/myproject/src mydoc
    creates the source code for all classes described in the directory mydoc and its subdirectories. The source codes will be saved in the directory /home/myname/workspace/myproject/src mydoc.

Limitations

Currently there are some known issues:

  • Import statements will not be generated.
  • Inner and anonymous classes will not be generated.
  • See tags will not be split, when there are several references.
